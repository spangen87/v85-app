# External Integrations

**Analysis Date:** 2026-04-05

## APIs & External Services

**ATG (Swedish Trotting Authority):**
- Service: ATG Racing Information API - Fetches V85/V75/V86/V64/V65 game data and horse statistics
  - Base URL: `https://www.atg.se/services/racinginfo/v1/api`
  - Endpoints consumed:
    - `GET /calendar/day[/{gameDate}]` - Lists available games for a specific date or today
    - `GET /games/{gameId}` - Retrieves full game with races and starters
    - `GET /horses/{horseId}` - Fetches horse career history and last 5 race results
  - Client: Fetch API (no SDK, raw HTTP in `lib/atg.ts`)
  - Rate limiting: Handled via retry logic; API may return empty `last_5_results` on rate-limit
  - Implementation: `lib/atg.ts` - `fetchAvailableGames()`, `fetchGame()`, `fetchHorseStarts()`

## Data Storage

**Databases:**
- Supabase (PostgreSQL)
  - Connection: Via `@supabase/supabase-js` client
  - Credentials: `NEXT_PUBLIC_SUPABASE_URL` and `NEXT_PUBLIC_SUPABASE_ANON_KEY`
  - Server operations: `SUPABASE_SERVICE_ROLE_KEY` for admin access via service client
  - Schema: `supabase/schema.sql` + migrations in `supabase/migration_v*.sql`
  - Key tables:
    - `games` - V85 omgångar (game_id, game_type, date, track)
    - `races` - Avdelningar per omgång (race_number, distance, start_method, start_time)
    - `starters` - Hästar i lopp (horse_id, odds, formscore, last_5_results JSONB, finish_position, finish_time)
    - `horses` - Horse master data (id = ATG horse_id, name)
    - `profiles` - User profiles (id = auth.uid, display_name)
    - `groups` - Sällskap/groups (invite_code, created_by, atg_team_url)
    - `group_members` - User-group membership
    - `horse_notes` - User annotations on horses (label, parent_id for replies)
    - `group_posts` - Forum posts per group and game

**File Storage:**
- Local filesystem only
  - PWA manifest and service worker: `public/` directory
  - Apple touch icon: `public/apple-touch-icon.png`
  - Service worker script: `public/sw.js` (generated by Serwist)

**Caching:**
- HTTP cache headers via Next.js `next` option in fetch:
  - `revalidate: 60` for ATG calendar (1-minute revalidation)
  - `revalidate: 0` for game data and horse stats (no caching, always fresh)
- In-memory caching: Horse starts history in request scope (not persisted)

## Authentication & Identity

**Auth Provider:**
- Supabase Auth
  - Implementation: Email/password authentication via `@supabase/supabase-js`
  - Client-side: `lib/supabase/client.ts` - `createClient()` using browser client
  - Server-side: `lib/supabase/server.ts` - `createClient()` and `createServiceClient()` for actions/routes
  - Sign in: `supabase.auth.signInWithPassword({ email, password })` in `app/login/page.tsx`
  - Protected routes: App Router pattern `app/(authenticated)/` - RLS policies handle access
  - Row Level Security: Authenticated users can only read shared tables; individual data tables use auth.uid foreign key
  - Session management: Cookies stored and refreshed via `@supabase/ssr`

**Protected Routes:**
- `app/(authenticated)/` - All child pages require valid Supabase session
- `app/login/` - Public login page (redirects to `/` on success)
- `app/join/[code]` - Public group join page

## Data Flow Integration Points

**Game Fetch Workflow:**
1. User clicks "Hämta omgång" button in `components/FetchButton.tsx`
2. POST to `app/api/games/fetch/route.ts` with `{ gameType, gameId }`
3. Route handler calls `lib/atg.ts` → `fetchGame()` to retrieve game from ATG
4. For each race, calls `fetchHorseStarts()` for last 5 results per horse
5. Calculates Composite Score via `lib/formscore.ts` → `calculateCompositeScore()`
6. Upserts to Supabase: games → races → horses → starters
7. Preserves prior results (`finish_position`, `finish_time`) from previous fetch

**Horse Analysis Workflow:**
1. Home page loads starters via Supabase RLS read policy
2. Components display formscore (FS) and Composite Score (CS) from starters.formscore
3. User can expand `HorseCard.tsx` to see:
   - Last 5 results from starters.last_5_results (JSONB)
   - Form indicators (3 best of last 5)
   - Value index calculation

**Authentication Workflow:**
1. Unauthenticated user directed to `/login`
2. Submit email/password to Supabase Auth
3. On success, session cookie set by `@supabase/ssr`
4. Redirect to `/` (authenticated layout)
5. All data fetches include auth token from cookie (automatic via server client)
